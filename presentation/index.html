### About me ![](images/PeterKirschner.jpg "Peter Kirschner") * name: Peter Kirschner * profession: IT software engineer * email: peter@kirschners.de * twitter: [@peterkir](http://twitter.com/peterkir) * github: https://peterkir.github.io/
### Development Mantra * bootstrap on OS * everything * locally - network is optional * portable (copy and paste) * java, eclipse, other tools * settings (~/.xyz, %USERPROFILE%) * eclipse (configuration, data, keyring, p2 data) * target products * be explicit and precise with versions for everything Note: Yes, I do know Internet ;-) BUT it is slow or not available!
### Initial development environment setup #### GOAL: getting java and bootstrap eclipse ready * install [Java SDK](http://www.oracle.com/technetwork/java/javase/downloads/index.html) (&(osgi.ee=JavaSE)(version>=1.7))[*](#/standards) * download [eclipse platform](http://download.eclipse.org/eclipse/downloads/) * unzip and launch bootstrap eclipse * **ready for Environment DevOps**
### prepare tooling and update setup * [create local mirror of required tools p2 repositories](http://help.eclipse.org/luna/topic/org.eclipse.platform.doc.isv/guide/p2_mirror.html?resultof=%22%70%32%2e%6d%69%72%72%6f%72%22%20) `mirror_repositories.xml` * [install with p2 director Eclipse IDE including bndtools](http://help.eclipse.org/indigo/index.jsp?topic=/org.eclipse.platform.doc.isv/guide/p2_director.html) * launch IDE and create a bndEvaluation workspace * launch IDE and create the product workspace
### Update development environment setup with new bndtools * install with p2 director Eclipse IDE including bndtools, ... into new location * create evaluation workspace with default bnd configuration project * compare/update bnd configuration project from eval to productive workspace

### common pitfalls * problem: mirroring a p2 repository results in .blobstore folder * solution: https://wiki.eclipse.org/Equinox/p2/p2.mirrorsURL#How_to_avoid_.blobstore_for_.2A.jar.pack.gz_files * problem: mirrored p2 repositories might contain inside references to other p2 repositories * solution: download references p2 repos and make remote reference relative
